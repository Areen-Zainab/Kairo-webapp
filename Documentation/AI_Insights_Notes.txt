AI models in use (agents) - UPDATED JAN 2026
- Summary Agent: Mistral AI mistral-small-latest (via Official Mistral SDK using MISTRAL_API_KEY env var). Switched from deprecated HuggingFace Inference API to direct Mistral API for reliability. Provides layered summaries: executive (2-3 sentences), detailed (full narrative), and bullet (quick scan). Fallback to Groq llama-3.3-70b-versatile if Mistral unavailable, then extractive summary as final fallback. Supports up to 32k tokens context. Alternatives: Groq llama-3.1-8b-instant for lower latency/cost with some quality drop.
- Decision Extraction Agent: Groq llama-3.3-70b-versatile. Excellent at schema-stable JSON extraction and reasoning about impact/context. Max_tokens set to 3000 to stay responsive; rate-limit handling with key rotation. Alternatives: Groq llama-3.1-8b-instant for faster latency; HF instruction-tuned FLAN-T5 XXL as hosted inference (would require aggressive chunking because of ~2k token max). Fallback is cue-phrase matching to guarantee output when API is unavailable.
- Action Item Agent: Groq llama-3.1-8b-instant (fast, cost-efficient) with optional HF google/flan-t5-large path. Max_tokens reduced to 1500 to keep latency low. Confidence carried through from model or set manually on fallback. Alternatives: Groq llama-3.2-3b-preview for even faster/cheaper at quality cost; Groq llama-3.3-70b-versatile for richer context when quality is critical. Pattern-based fallback guarantees a result even without LLM.
- Topic Segmentation Agent: Groq llama-3.3-70b-versatile with retries/key rotation. Chosen for grouping discourse across long context and emitting ranked topics. Alternatives: Groq llama-3.1-8b-instant for speed; classical TF-IDF/LDA for offline fallback (current fallback is paragraph splitting with heuristics).
- Sentiment Analysis Agent: Groq llama-3.3-70b-versatile. Low max_tokens (1000) keeps latency modest. Alternatives: DistilBERT/roberta-sentiment models (fast, on-prem) for cheaper runs; Groq llama-3.1-8b-instant for speed; rule-based lexicons (current fallback) for zero-cost continuity.
- Participant Analysis Agent: Groq llama-3.3-70b-versatile over diarized JSON converted to text, plus computed speaking-time stats merged post-call. Alternatives: Groq llama-3.1-8b-instant for speed; statistical-only mode (already implemented) for offline runs; specialized engagement models (pyannote speaker analytics) if GPU is available.
- Orchestration: run_all() executes text agents in parallel; participant analysis runs when JSON is provided; summary uses outputs from other agents as context to boost coherence and reduce hallucinations.

Transcription model
- Current stack: WhisperX (faster-whisper backend) model_size=small, device=CPU, compute_type=int8. Selected for acceptable WER with low CPU footprint and no GPU dependency; int8 keeps RAM/VRAM low and startup lighter. Diarization via WhisperX pipeline (Silero VAD; pyannote disabled). Typical throughput: faster-whisper small int8 on CPU ~0.6-1.2x real-time; chunk0 load adds 10-30s if model not warm.
- Planned migration: faster-whisper direct (see FASTER_WHISPER_MIGRATION_PLAN.md) to reduce latency and control compute_type (int8/float16) and device selection; retain ffmpeg ingestion. Alternatives: Groq audio (whisper-large-v3 turbo) for cloud speed; Deepgram Nova-2 or AssemblyAI for hosted ASR; OpenAI Realtime API for live streaming; on-prem Nvidia Riva for GPU environments. Token limits are not the constraint for ASR; latency and diarization quality are.

AI insights techniques and fallbacks
- Parallel agent execution for text-based tasks; participant analysis runs separately when JSON is present.
- Context reuse: summary consumes topics, decisions, actions, sentiment, and participant stats to keep outputs coherent and reduce contradictions.
- Fallback tiers per agent: Hugging Face hosted inference (where configured) → Groq primary → heuristic/pattern/statistical extraction. Confidence is lowered on heuristic paths.
- Transcript conversion: diarized JSON → rich text (with speaker/time) for agents; JSON is also kept for participant stats; converter validates structure and can output text-only when needed.
- Database guardrails: single-run check per meeting, partial saves allowed, agent failures logged without aborting others.

Latency and delay notes (transcription + insights)
- Current pain: chunk0 waits on model warmup (10-30s) plus remux (~0.5-1s) plus ASR (~2-5s) → 15-40s. Subsequent chunks: 6-15s due to cumulative-blob extraction and sequential queueing.
- Fixes already noted: preload model globally; remove MP3 conversions; duplicate prevention; retry improvements.
- Remaining high-impact fixes: readiness check before bot join; switch browser to true 3s incremental chunks to drop extraction cost; process 2-3 ASR jobs in parallel with shorter timeouts; push updates via WebSocket/SSE to remove 3s polling delay; optionally move to faster-whisper direct with GPU/int8_float16 when available.
- AI Insights latency: Groq calls in parallel; each call capped with max_tokens (1k-3k) to keep responses under ~5-12s. Typical full run 30-120s depending on transcript length and queueing. Chunking in summary reduces token load for long meetings (~1.5k-word chunks with 300-word overlap).

Confidence scoring
- Per-agent: models return confidence fields in their JSON (actions/decisions set by model or default 0.8; sentiment uses model confidence; summary uses fixed 0.6-0.85). Fallback heuristics downgrade to ~0.5 to signal lower reliability.
- Participant analysis merges model judgments (engagement/sentiment) with measured speaking-time percentages; speakingTime and speakingTimeSeconds are deterministic, while engagement/sentiment rely on model output.
- For multimodal extensions: compute modality-specific confidence (ASR quality from VAD/word error proxies, vision/OCR confidence, LLM reasoning confidence) and combine with calibrated weights rather than averaging blindly. Prefer source-level calibration (e.g., ASR log-probs, vision detector scores) and propagate per-item scores through downstream aggregation.

Why the current choices fit
- Groq chat models (llama-3.3-70b-versatile primary, llama-3.1-8b-instant for speed-critical tasks) balance long-context capacity (128k tokens), excellent JSON stability, and strong rate-limit handling with multi-key rotation; they are 100% free and faster than frontier GPT-4-class models while meeting quality needs for meeting summarization/extraction. Llama-3.3-70b provides superior instruction following and reasoning compared to previous openai/gpt-oss-120b model.
- WhisperX small int8 on CPU minimizes operational friction (no GPU requirement) while delivering diarization and acceptable accuracy; aligns with self-hosting needs and avoids vendor lock-in. Migration to faster-whisper direct keeps these benefits with better latency.
- Fallbacks ensure resilience: HF inference where configured, then deterministic heuristics so pipeline always returns structured data instead of failing hard.
- Layered summaries provide flexibility: executives get 2-3 sentences, participants get detailed narratives, quick reviewers get bullet points - all from one generation pass with minimal added latency.
